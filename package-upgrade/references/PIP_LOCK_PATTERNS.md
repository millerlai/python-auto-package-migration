# Pip Lock 檔案模式說明

## 常見的 Pip 專案結構模式

### 模式 1: pip-tools (推薦)

**檔案結構**:
```
project/
├── requirements.in          # 約束檔案 (手動編輯)
└── requirements.txt         # Lock 檔案 (自動產生)
```

**特徵**:
- 有 `requirements.in` 檔案
- `requirements.txt` 是由 `pip-compile` 自動產生
- `requirements.txt` 開頭通常有註解: `# This file is autogenerated by pip-compile`

**如何更新**:
```bash
# 1. 編輯 requirements.in
vim requirements.in

# 2. 重新編譯
pip-compile requirements.in

# 3. 安裝
pip-sync requirements.txt
```

**檢測方法**:
```bash
if [ -f "requirements.in" ]; then
    echo "使用 pip-tools"
fi
```

---

### 模式 2: 自定義 Lock 檔案

**檔案結構**:
```
project/
├── requirements.txt         # 約束檔案 (手動編輯)
└── requirements.lock        # Lock 檔案 (pip freeze 產生)
```

**常見 lock 檔案名稱**:
- `requirements.lock`
- `requirements.txt.lock`
- `requirements-lock.txt`
- `requirements/production.lock`
- `requirements/base.lock`

**如何更新**:
```bash
# 1. 編輯 requirements.txt
vim requirements.txt

# 2. 安裝並產生 lock
pip install -r requirements.txt
pip freeze > requirements.lock

# 3. (可選) 從 lock 檔案驗證
pip install -r requirements.lock
```

**檢測方法**:
```bash
# 檢查常見的 lock 檔案名稱
for lock_file in \
    "requirements.lock" \
    "requirements.txt.lock" \
    "requirements-lock.txt" \
    "requirements/production.lock"; do
    if [ -f "$lock_file" ]; then
        echo "找到 lock 檔案: $lock_file"
        break
    fi
done
```

### 模式 3: 多環境 + Lock

**檔案結構**:
```
project/
├── requirements/
│   ├── base.txt             # 基礎依賴
│   ├── base.lock            # 基礎 lock
│   ├── production.txt       # 生產依賴
│   ├── production.lock      # 生產 lock
│   ├── development.txt      # 開發依賴
│   └── development.lock     # 開發 lock
```

**如何更新**:
```bash
# 1. 編輯對應的 .txt 檔案
vim requirements/base.txt

# 2. 產生對應的 .lock
pip install -r requirements/base.txt
pip freeze > requirements/base.lock
```

---

### 模式 4: 無 Lock (直接精確版本)

**檔案結構**:
```
project/
└── requirements.txt         # 直接使用精確版本 (==)
```

**特徵**:
- 沒有 `.lock` 檔案
- 沒有 `requirements.in`
- `requirements.txt` 中都使用精確版本 (`==`)

**如何更新**:
```bash
# 1. 直接編輯 requirements.txt
vim requirements.txt

# 2. 安裝
pip install -r requirements.txt
```

## 自動檢測流程

### 檢測優先級

1. **檢查 pip-tools**: 是否有 `requirements.in`
2. **檢查自定義 lock**: 是否有 `*.lock` 檔案
3. **預設**: 無 lock 模式

### detect_env.sh 實作

```bash
PKG_MANAGER="pip"
PIP_LOCK_FILE=""
HAS_PIP_TOOLS="false"

# 1. 檢查 pip-tools
if [ -f "requirements.in" ]; then
    HAS_PIP_TOOLS="true"
    LOCKFILE="requirements.txt"
    PIP_LOCK_FILE="requirements.txt"

# 2. 檢查常見 lock 檔案
elif [ -f "requirements.lock" ]; then
    PIP_LOCK_FILE="requirements.lock"
elif [ -f "requirements.txt.lock" ]; then
    PIP_LOCK_FILE="requirements.txt.lock"
elif [ -f "requirements-lock.txt" ]; then
    PIP_LOCK_FILE="requirements-lock.txt"
elif [ -f "requirements/production.lock" ]; then
    PIP_LOCK_FILE="requirements/production.lock"

# 3. 無 lock
else
    PIP_LOCK_FILE=""
fi
```

## 在 Skill 中的處理

### Phase 0: 環境偵測

輸出 JSON 包含:
```json
{
  "pkg_manager": "pip",
  "pip_lock_file": "requirements.lock",
  "has_pip_tools": false
}
```

### Phase 5.3: 更新依賴

根據檢測結果:

#### 如果 `has_pip_tools = true`:
```bash
# 自動執行
pip-compile requirements.in
pip-sync requirements.txt
```

#### 如果 `pip_lock_file` 不為空:
```
暫停並詢問使用者:

"偵測到專案使用 lock 檔案: requirements.lock

請確認如何產生 lock 檔案:
[1] 使用 pip freeze (標準方式)
[2] 使用專案自定義腳本
[3] 我會手動處理

請選擇: "
```

#### 如果兩者都沒有:
```bash
# 直接編輯並安裝
# 編輯 requirements.txt
pip install -r requirements.txt
```

## 常見問題

### Q: 如何判斷 requirements.txt 是約束檔案還是 lock 檔案?

**A**: 檢查是否有 `requirements.in`:
- 有 `requirements.in` → `requirements.txt` 是 lock 檔案
- 沒有 `requirements.in` → `requirements.txt` 是約束檔案

### Q: 如果專案使用自定義腳本產生 lock 怎麼辦?

**A**: 在 Phase 5.3 詢問使用者:
```
偵測到 lock 檔案,但不確定產生方式。

常見的產生方式:
1. make lock
2. ./scripts/generate-lock.sh
3. tox -e lock
4. pip freeze > requirements.lock

請告訴我你的專案使用哪個命令,或選擇 [3] 手動處理。
```

### Q: 多環境專案 (dev/prod) 怎麼處理?

**A**: 檢測 `requirements/` 目錄:
```bash
if [ -d "requirements" ]; then
    # 列出所有 .txt 和 .lock 檔案
    # 詢問使用者要更新哪個環境
fi
```

## 最佳實踐建議

### 推薦: 使用 pip-tools

**優點**:
- ✅ 明確區分約束 (.in) 和鎖定 (.txt)
- ✅ 自動解析依賴樹
- ✅ 可以只升級特定套件
- ✅ 有標準工具 (`pip-compile`, `pip-sync`)

**安裝**:
```bash
pip install pip-tools
```

**遷移**:
```bash
# 1. 建立 requirements.in (只列直接依賴)
cat > requirements.in <<EOF
requests>=2.32.0
flask>=3.0.0
EOF

# 2. 產生 lock 檔案
pip-compile requirements.in

# 3. 安裝
pip-sync requirements.txt
```

### 如果不想用 pip-tools

至少使用自定義 lock 檔案:
```bash
# 1. 維護 requirements.txt (約束)
# 2. 產生 requirements.lock (lock)
pip install -r requirements.txt
pip freeze > requirements.lock

# 3. CI/CD 使用 lock
pip install -r requirements.lock
```
